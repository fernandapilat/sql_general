# Data Modeling Course | Case Study: Flexempresta

This repository serves as a centralized hub for my data modeling journey, documenting the complete lifecycle of a database—from initial business requirements to a high-performance physical implementation.

## Learning Path

The project followed a structured engineering workflow to ensure data reliability and scalability:

| Module | Phase | Key Deliverables |
| :--- | :--- | :--- |
| **01** | **[Conceptual Modeling](./01_conceptual_model/)** | Business rules analysis, entity identification, and ER Diagrams (DER). |
| **02** | **[Logical Modeling](./02_logical_model/)** | Relational mapping, Primary Keys (PK), and Foreign Keys (FK) definition. |
| **03** | **[Normalization](./03_normalization/)** | Optimization through 1NF, 2NF, and 3NF to eliminate redundancies. |
| **04** | **[Physical Modeling](./04_physical_model/)** | MySQL implementation, DDL scripts, Constraints, and Performance Tuning. |

---

## Tooling & Technical Skills

* **CASE Tool:** Visual Paradigm (ERD & Relational Diagramming).
* **DBMS:** MySQL (Implementation, Forward & Reverse Engineering).
* **Documentation:** Structured technical documentation in Markdown.
* **Methodology:** Transitioning business logic from flat files (Excel) to professional RDBMS environments.

---

## Key Achievements & Implementation Details

* **Data Integrity:** Enforced business rules at the database level using `CHECK`, `UNIQUE`, and `NOT NULL` constraints.
* **Performance Optimization:** Strategic indexing to improve query response times for analytical workloads.
* **Workflow Mastery:** Practical application of Forward Engineering (Model to DB) and Reverse Engineering (DB to Model).
* **Sustainability:** Designing data structures that allow for safe schema evolution (Synchronize Model).
